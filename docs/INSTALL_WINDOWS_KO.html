Claude Stream Deck 플러그인

Windows 재설치 및 복구용 설치 안내입니다.

릴리스 ZIP 구성: Claude Deck Companion Setup *.exe, com.hanbroz.claude-usage.streamDeckPlugin, Install.cmd, SHA256SUMS.txt

1. 재설치 전에 보관할 파일

  1. 릴리스 ZIP 파일 전체를 USB, NAS, 클라우드 저장소 등에 보관합니다.
  2. Stream Deck 앱에서 Settings > Profiles > Backup All > Create Backup으로 .StreamDeckProfilesBackup 파일을 따로 보관합니다.
  3. Code Start에서 사용하는 프로젝트 폴더도 백업합니다. 드라이브 문자나 경로가 달라지면 복구 후 버튼 설정에서 새 경로를 지정해야 합니다.

프로필 백업에는 버튼 배치와 설정이 들어가지만, 비공개 플러그인의 설치 파일은 포함되지 않습니다. 릴리스 ZIP을 함께 보관하십시오.

2. Windows 설치 순서

  1. Elgato 공식 사이트에서 Stream Deck 7.1 이상을 설치하고 한 번 실행합니다.
  2. PowerShell에서 Claude Code를 설치하고 로그인합니다.
winget install Anthropic.ClaudeCode
claude --version
claude
  1. 릴리스 ZIP을 압축 해제한 뒤 Install.cmd를 실행합니다.
  2. Install.cmd는 먼저 Claude Deck Companion Setup *.exe를 실행합니다.
  3. Companion 설치 프로그램은 wt.exe를 확인합니다. 없으면 다음 WinGet 명령을 자동 실행하고 다시 확인합니다.
winget install --id Microsoft.WindowsTerminal -e --source winget --silent --accept-package-agreements --accept-source-agreements

WinGet이 없거나 회사 정책, Store 차단, 네트워크 문제로 Windows Terminal을 확인할 수 없으면 설치가 중단됩니다. 이 경우 Microsoft Windows Terminal 설치 안내를 따라 수동 설치한 뒤 다시 실행하십시오.

  1. Companion 파일 설치가 끝나면 설치 프로그램이 번들된 com.hanbroz.claude-usage.streamDeckPlugin을 자동으로 엽니다.
  2. Stream Deck 설치 확인 창에서 설치를 승인합니다.
  3. 프로필 백업이 있으면 .StreamDeckProfilesBackup 파일을 더블클릭하고 Restore를 선택합니다.
  4. 프로필 백업이 없으면 Claude 그룹의 버튼을 다시 배치하고 Code Start에 프로젝트명과 폴더를 입력합니다.
  5. Code Start 버튼을 한 번 누릅니다. Claude Deck Companion이 열리고 Claude Code가 실행되며 상태 브리지가 자동 설치됩니다.

3. 정상 동작 확인

  • Code Start: 지정한 폴더의 Companion 창이 열리고 프로젝트명, 현재 모델, 컨텍스트 사용량 막대가 표시됩니다.
  • 세션 재개: 같은 버튼과 같은 폴더를 다시 열면 마지막으로 관측한 같은 Claude 세션 ID를 사용해 재개합니다.
  • 터미널 열기: Companion의 터미널 열기 기능은 wt.exe -d <프로젝트 폴더>만 실행하며 Claude를 새로 시작하지 않습니다.
  • 프로젝트 탐색기: 파일 열기, 폴더 펼치기, 새 파일/폴더 만들기는 설정한 프로젝트 루트 안에서만 동작합니다.
  • Usage 버튼: Claude Code에서 메시지를 한 번 처리하면 5시간 및 7일 사용량이 표시됩니다.

4. 재설치 후 달라지는 항목

  • 재설치 전에 실행 중이던 Claude 프로세스와 Code Start 캐시는 복구되지 않습니다. Code Start로 새 Companion 세션을 시작하면 다시 표시됩니다.
  • Claude 로그인 정보는 다시 인증해야 할 수 있습니다.
  • 프로젝트 경로가 바뀌었다면 각 Code Start 버튼에서 폴더를 다시 선택하고 Save Settings를 누릅니다.

5. 문제 해결

  • Companion 설치가 Windows Terminal 단계에서 중단되면 공식 안내로 Windows Terminal을 설치한 뒤 Install.cmd를 다시 실행합니다.
  • 플러그인이 보이지 않으면 Stream Deck 앱을 완전히 종료했다가 다시 실행합니다.
  • Claude가 실행되지 않으면 PowerShell에서 Get-Command claude와 claude --version을 확인합니다.
  • 버튼이 STARTING에 머무르면 프로젝트 폴더가 존재하는지 확인하고 버튼을 다시 누릅니다.
  • 설치 후 문제가 생기면 Stream Deck 설정에서 Claude 플러그인을 제거한 뒤 보관한 릴리스 ZIP으로 다시 설치합니다.
